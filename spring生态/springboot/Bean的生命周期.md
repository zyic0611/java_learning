# Bean的生命周期

实例化-属性赋值-初始化-销毁

1. 实例化:
   - 使用反射机制new了一个对象
   - 只是一个半成品 内部没有属性
2. 属性赋值
   - 使用DI 把需要的依赖塞进属性
3. 初始化
   - 初始化前置处理器
   - 自定义初始化方法 @PostConstruct
   - 初始化后置处理器
4. 销毁 
   - spring容器关闭 
   - 或者@PreDestroy方法

## 初始化

### 自定义初始化方法 

传统java中 如果项目提前需要加载xx缓存 就比如手动调用哦缓存的初始化方法。

但是使用@PostConstruct注解 可以确认在这个bean的依赖装配好之后 自动执行里面的方法。

在 资源预热 初始化连接池 启动后台定时线程时常用。



### 前后置处理器

可以理解为是两个安检门 在@PostConstruct之前和之后 都需要通过安检门检查。

1. 前置处理器
   - 一般做一些底层的准备 比如spring内部检查这个Bean有没有实现某些特殊的接口 有的话 就在这个阶段把上下文对象给Bean
   - 基本都是框架底层在使用
2. 后置处理器

大部分注解都是在这里实现 。你想想执行完自定义初始化后 就需要开始执行方法了。方法前一般都有注解，所以后置处理器就是在这个时候开始的。

在这产生AOP动态代理：

创建普通bean 只是java对象 不懂的数据事务。

当普通bean走到了后置处理器这个安检门 发现有事务 

则帮这个bean生成一个代理对象 放进spring容器 然后把java普通bean隐藏起来  帮忙执行事务逻辑。
