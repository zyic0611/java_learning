# TCP保证数据包不丢失

1. 序列号与确认答应： tcp给发送的每一个字节都打上了序列号 由于网络路径不同 先发的数据可能后到 接收端拿到包后 在自己tcp操作系统内核层面 通过序列号凭借排序 解决了乱序问题。 并且每接收到一个包 会回复一个ack 确认答应 告诉发送端  序列号多少以前的数据我都收到了 下次从序列号+1开始发
2. 重传机制：发送端发送图片后 迟迟没收到ack
   - 超时重传：设置定时器 时间到了没收到ACK 直接重发
   - 三次重复ACK 接收端发现少了一个包 则会连续发3个重复ACK 发送端收到就直接重发丢失的包
3. 滑动窗口：解决了接收方接受了过多的数据 导致后面的包丢失 接收端会在返回的ACK上写上自己的缓冲区空间大小 发送端根据窗口控制自己的发送速度 流量控制
4. 拥塞控制 ：解决网络卡顿问题 tcp会采取慢启动 拥塞避免等算法 探查网络堵塞情况 动态调整发包速度 避免包丢失

序列号防乱序，ACK 加重传防丢包，滑动窗口防撑爆接收端，拥塞控制防搞崩整个网络。
