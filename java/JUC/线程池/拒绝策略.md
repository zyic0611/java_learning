# 拒绝策略

AbortPolicy 默认 直接抛出拒绝执行异常

CallerRunsPolicy 谁提交的业务 让谁执行

DiscardPolicy 静默丢弃任务

DiscardOldestPolicy 丢弃阻塞队列最老的一个任务 然后重新尝试提交。



**业务抉择：** 在你的智慧环卫业务中，如果是“高频上传垃圾图片进行 AI 推理”这个非强一致性的业务场景，你会选择哪种默认策略，或者你会如何自定义一个拒绝策略来优雅降级？



高频 不强一致。

可以去实现抛出异常接口 当被拒绝的时候 不掉算法API 把图片的URL和任务ID写入Mysql的一直延迟补偿表或者Redis的list里。

直接返回 系统繁忙 预计处理时间延长。

高峰过了 在用定时任务处理。
