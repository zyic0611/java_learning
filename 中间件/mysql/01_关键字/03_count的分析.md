# count

## count性能分析

count的性能 取决于扫描哪个索引 。

### count(*) 

 count(*) ≈ count(1) 

优化器会统一处理 都表示统计所有行。

count(*) 会选择 成本最低的执行计划。

通常表现为 扫描占用空间最小的索引 一般是二级索引 因为二级索引比比主键（簇状索引）更小。

**不需要判断null**

### count(字段)

如果该字段没索引 需要全盘扫描 还需要**判断null** 更慢

如果该字段有索引 性能不错。 如果该索引更小 性能甚至> count(*)

### 性能核心

取决于是否走索引  索引大小 是否回表/判断null
