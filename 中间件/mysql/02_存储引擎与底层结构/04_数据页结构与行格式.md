# 数据页结构 和 行格式

## 行记录的链表格式

数据页的内部 每一行插入的数据 在物理上 并不一定连续 但是在逻辑上 通过单向链表 按主键串联在一起  



问题： 如果在单向链表里找一条数据 时间复杂度为on 对于16kb的一页来说 效率很低



### 页目录

innodb把页内所有记录分为很多组 一般4-8条记录为一组

每组的最后一条记录 一般是组里最大的 的物理地址偏移量 会被提取出来 放在页目录里 提取出来的值称为槽



核心方法：

因为槽是按主键顺序连续存放的，所以 InnoDB 在一个页内查找数据时，**绝不是遍历链表，而是直接在 Page Directory 中使用二分查找法 (Binary Search)**。找到对应的槽后，再回到链表里遍历那区区几条记录。这就把页内查询的效率提升到了 $O(\log N)$。
