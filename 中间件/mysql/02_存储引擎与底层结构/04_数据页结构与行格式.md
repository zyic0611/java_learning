# 数据页结构 和 行格式

## 行记录的链表格式

数据页的内部 每一行插入的数据 在物理上 并不一定连续 但是在逻辑上 通过单向链表 按主键串联在一起  



问题： 如果在单向链表里找一条数据 时间复杂度为on 对于16kb的一页来说 效率很低



### 页目录

innodb把页内所有记录分为很多组 一般4-8条记录为一组

每组的最后一条记录 一般是组里最大的 的物理地址偏移量 会被提取出来 放在页目录里 提取出来的值称为槽



核心方法：

因为槽是按主键顺序连续存放的，所以 InnoDB 在一个页内查找数据时，**绝不是遍历链表，而是直接在 Page Directory 中使用二分查找法 (Binary Search)**。找到对应的槽后，再回到链表里遍历那区区几条记录。这就把页内查询的效率提升到了 $O(\log N)$。



## 行记录格式

### compact格式

A行头：

变长字段长度列表 ：记录varchar字段的实际长度

null值列表：为了节省空间 mysql使用bitmap来记录哪一列为null 允许null列实际不占用数据空间

记录头信息 包含很多标志位  deleted_flag 删除标记 、next_record 指向下一条记录的指针



B隐藏列：

是MVCC和事务回滚的底层基石

1 DB_ROW_ID  如果没有主键&uniqueidx 用它来自动生成聚簇索引

2 DB_TRX_ID 事务ID 记录最后一次插入或修改这一行记录的事务ID 用于MVCC

3 DB_ROLL_PTR 回滚指针 指向这行记录的上一个版本（放在undolog中 发生并发读的时候 如果当前版本不可见 顺着指针去undolog中寻找历史版本



### 行溢出机制

一个页只有16kb 16384B 但是mysql规定一行最多可以65535字节

compact格式： 在当前行的真实数据部分保留768个字节 使用20字节指针指向溢出页 BLOB PAGE

dynamic格式 真实数据不保留 全部放在溢出页 这样子一个页能存的行数更多。



溢出页： 可能是1页 也可能是多页 多页采用指针连接。
