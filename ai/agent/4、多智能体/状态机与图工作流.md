# FSM & Graph

状态 state

也就是 全局上下文。 在图工作流中 必须有一个贯穿始终的全局对象 。 理解为所有agents共享的内存。（或者后端系统中 一个包含当前执行进度的DTO对象/Redis缓存）

作用： 每当一个agent执行完任务 必须且只能把结果更新到这个state中。 下游的agent从state中读取数据进行下一步工作。



节点 nodes

执行单元。 图中的每一个节点就是具体的动作。 他可以是一个agent 也可以是Tool。



边 edges

路由与流转边界。 决定了系统从一个节点执行完毕 下一步去哪里。

普通边 ：强制的先后顺序

条件边： 系统的分叉口， 由确定性代码逻辑决定 比如ifelse 而不是由llm瞎猜判断。





dag与有环图

dag是有向无环图 在多智能体架构中 需要反思与重试 所以必须需要有环图。



## 优点

1. 能力解偶 不同的功能可以用不同能力的模型
2. 可干预性和断点续传 基于状态机 系统可以在任何一个节点暂停 
3. 确定性兜底 图工作流明去了llm能波及的边界 产生了幻觉 条件边也会拦截
