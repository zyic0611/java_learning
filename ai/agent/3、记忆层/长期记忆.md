# 长期记忆

短期记忆一般存在内存中 长期记忆就是让agent有跨session的记忆 哪怕重启系统 还记得聊天记录。

核心在于该存什么 不该存什么 什么都存 就上去了长期记忆的意义了。

## 向量数据库外挂

把过去的对话切片 chunking， 做Embedding化后存入 向量数据库 当用户有新输入的时候 做语义相似度检索（KNN） 把最相关的历史片段召回。

重点：

- 时间问题： 传统RAG对时间不敏感 比如存了两条信息 “我喜欢喝冰美式”(3月) 和 “我戒咖啡了，现在只喝茶”(4月)。如果用户问“给我点杯喝的”，单纯依靠语义检索，两条都会被召回，模型会精神分裂
  - 解决： 在向量记录打上Timestamp的元数据 并且在召回的时候做时间衰减过滤。
- 事实碎片化： 对话被切成chunk后 原有的上下文语境可能被破坏。
  - 解决： 使用父子文档检索 或者在chunk的时候保留对话上下文前缀。





## 实体/图谱记忆

利用 LLM 的信息提取（IE）能力，从对话中抽取出结构化的事实（如 User Profile） 一般是偏好。，存入 KV 数据库或图数据库（如 Neo4j）。形式往往是 `(User, Likes, Deep_Roast_Coffee)` 三元组。

问题：

- curd的冲突管理（状态更新）：vectordb 只能做append 很难做update。 而实体记忆的价值在于 状态变更。 
  - 解决： 用户的一个行为 agent需要识别出这个是一个新偏好 还是旧偏好的更新。 通常以来llm的路由判断 ： 来决定写入新实体 还是覆写旧实体。
- 延迟提取： 并不是在每一轮对话都去提取实体 效率低 成本高。 
  - 解决：工程上通常采用 批处理or事件驱动（对话监测到明显的偏好表达）

## 记忆衰减

什么都往数据库存 噪音多 干扰模型决策。 存的少 失去了长期记忆的意义。

给每条记忆加一个时间权重 越久远的记忆权重越低。 

记忆的相关性分数 = 语义相似度 x 时间衰减因子，时间衰减因子随着时间推移逐渐降低。

## 记忆分类

1. 程序性记忆： 存怎么做事的经验  不存某个具体的事实  把agent做事的方法存储下来 让下次不用从头摸索
2. 语义记忆： 存储的是事实行知识 xx是什么
3. 情景记忆：存储的是某个具体的经历。    用户上次说的 agent采取的结果 查出的结果 可以直接返回 直接考虑在内
