# 数据结构

## 宏观

1. String ：最基本K-V 适合缓存对象，计数器INCR 分布式锁 SETNX。
2. Hash ：键值对集合，用于存储对象信息，支持单对象修改：把一个对象的每一个属性用hash存储，可以修改每一个属性的值。如果一个对象用String存储的话，首先存储进去就得序列化。想修改一个属性，就得把对象从redis里反序列化出来，修改后，再序列化后存储到reids里。
3. List:  双向链表 适用于简单的消息队列 最新动态时间线 TimeLine
4. Set： 无序的去重集合 适用于社交场景求交/并集。 共同关注，共同好友
5. ZSet：带权重的有序集合 内部维护一个score字段。适用于排行榜，延时队列。

## 微观

1. SDS 简单动态字符串 代替C语言原生字符串。
   - C语言自带String 获取长度需要遍历一遍on，稍微加点内容就容易导致内存溢出
   - SDS 再字符串加头部 记录了已经用的长度和没用的长度
   - 找长度 o1 提前预分配空间 不会溢出
2. ZipList/ListPack 压缩列表
   - 在Hash/Zset 数据量很小的时候，也就是只有两三个键值对的时候，去构建一个复杂的哈希表活着树，很浪费内存
   - ZipList 使用一块坚硬的连续内存，把数据放在一起存储，省去了指针的内存开销。
   - 省内存，但是修改慢，需要挪动大量数据，所以只在小数据的时候用
3. SkipList 跳表 Zset使用
   - 问题：想做百万人的排行榜，需要快速查到排名，以及可以随时插入新人。 数组：插入太慢   链表：查询太慢  红黑树：代码复杂 范围查询很难，只能单独查询效率高
   - 跳表：它在普通链表的基础上，**抽取出多层“索引”**。
   - 不仅查询速度堪比红黑树logn，而且**实现极其简单，天然支持完美的范围查询**
