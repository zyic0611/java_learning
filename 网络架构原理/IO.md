# IO

1. 什么是i/o 

   - 只要是CPU&&memory与外界打交道 统称IO
   - 和本地硬盘打交道叫 磁盘I/O 
   - 通过网线/WIFI和另一台电脑打交道 叫做网络I/O

2. BIO：

   - Blocking i/o 同步阻塞I/O
   - 服务器线程 服务 一个客户端 在BIO模式下 必须只服务这个客户端 在服务期间内 一直阻塞着 无法服务其他客户端
   - 如果同时来1000个客户端 则必须同时配置1000个线程 非常浪费资源

3. NIO

   - Non-blocking I/O
   - 轮询 不会阻塞在一个客户端上。 线程发起I/O请求后 如果数据没有准备好 会立刻返回一个状态，线程可以去执行其他逻辑 不会傻等

   三大核心组件

   1. buffer  缓冲区 nio面向块/缓冲区编程 数据都是放进buffer 相比较bio面向流 效率更高
   2. channel 代替了bio stream 双向的 可以支持读写
   3. selector 是一个多路复用器 一个selector（单线程  可以同时监听上万个channel 哪个channel数据准备好了 就去处理哪个通道

   效果：

   - 极少的线程 配合sellector的线程监听机制 就可以管理数以万计的客户端链接
